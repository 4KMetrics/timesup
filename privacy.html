Privacy Policy

Time's Up Browser Extension — Effective February 2026

Time's Up respects your privacy completely.

Data Collection

None. Time's Up does not collect, transmit, or share any data. Period.

  • No analytics or telemetry
  • No external network requests
  • No user tracking
  • No advertising
  • No account required

Data Storage

All data is stored locally in your browser using chrome.storage.local:

  • Configuration — your blocking rules, time limits, categories, and preferences
  • Usage data — time spent per site per day (kept for 7 days, then auto-deleted)
  • Snooze state — temporary bypass records (cleared daily)

This data never leaves your browser. It is not synced to any server, cloud service, or third party.

Permissions Explained

PermissionWhy It's Needed
storageSave your config and usage data locally
alarmsPeriodic time tracking (1-minute tick) and daily reset
tabsTrack which site is active to count time
activeTabAccess the current tab's URL for domain matching
declarativeNetRequestBlock sites by redirecting to the blocked page
idlePause time tracking when you're away from the computer
scriptingInject doomscroll detection script into configured sites only
host_permissions: <all_urls>Required for declarativeNetRequest to block any site

Third Parties

Time's Up does not share data with any third parties. There are no third-party SDKs, libraries, or services embedded in the extension.

Changes to This Policy

If we update this privacy policy, the changes will be reflected here with an updated effective date.

Contact

For privacy questions, reach out via the support link on the Chrome Web Store listing.
